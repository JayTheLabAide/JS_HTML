Simple Styling

HTML and CSS are used side-by-side, and this can be confusing, but many keywords are similar. CSS usually is indicated by keyword STYLE, and everything in simple angle-bracket tags is HTML.

List Tips

  • Use H or LABEL to title a list.
  • Use LI for every item.
  • Use tabs to format lists, even though that won't show up when the HTML is rendered.
  • Don't put spaces after the opening LI tags.

Div and Span

Before Div
Div is used when you want this section to be on its own lines (even if the HTML source has stuff on the same line)
After Div. Before Span Span is used when you want a section designated to share lines with what's around it. After Span.
If opening and closing divs gets confusing, line break tag BR can be used instead. Use two to skip a line.

Divs can be given a specific width in pixels or an element-relative width in percentages. It is a bit strange, but divs are usually centered horizontally by specifying the margin as 0 on top and bottom, and "auto" for left and right.

CSS classes can be used to give Ps and DIVs and SPANs and lists consistent themes. These are often declared in STYLE sections. Inside this section CSS rules apply, like using : instead of = or ending each line with ;
These style sections can appear anywhere in the text, including at the bottom, and most browsers will still apply that style everywhere. To be safe, though, most often times STYLE sections are put in the HEAD section of the page, or at the top of the BODY section (but not inbetween). The STYLE section in the HTML source after this line of text is used to modify colors and borders in the elements that follow.
