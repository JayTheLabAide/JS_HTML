Alignment Methods

Alignment is a property you can alter in a few different elements which all act as text boxes. There are two reliable ways to do this. The first uses just HTML. The second is used with the style property, and therefore uses CSS syntax. Why are there two systems for doing nearly the same things? I don't know, it's dumb, and it will probably end up being replaced and simplified by something more sensible in the future.

Method 1

Use align="whatever" within an element's tag to adjust alignment in that element.

<code align="left">Text between CODE tags gets aligned. (CODE tags just look like code, they don't represent any specific programming language).</code>

<div align="center">Text between DIV tags gets aligned</div>

<h3 align="right">Even text between H tags gets aligned</h3>

<p align="justify"> This text should fill the width of the text box. All these different alignment options can be used in P, DIV, H, ARTICLE, CODE, and most tags related to text. Unfortunately this alignment property does not work within a SPAN tag, which you will learn more about later. The reason this doesn't work in SPAN is because those are only as big as the text inside the opening and closing tags, and so there is no space on either side of the text to align. CODE elements, which use monospaced fonts by default, can't use justified alignment because that violates the fixed spacing of the font. </p>

Method 2

Using style="text-align: something" to in-line CSS.

<p style="text-align: left">Text between P Tags</p>

<p style="text-align: center">Text between P Tags</p>

<p style="text-align: right">Text between P Tags</p>

Div and Span

[Text Before DIV]
DIV is used when you want this section to be on its own lines, in its own text box (even if the HTML source has stuff on the same line as the DIV tags)
[Text After DIV] [Text Before SPAN] SPAN is used when you don't want text to be in a separate paragraph, but instead want to alter or highlight just one section of a paragraph. [Text After SPAN]
If opening and closing DIV or P tags just to put things in their own paragraphs gets confusing, you can always just use the line break tag <BR> all over the place Use two to skip a line, like this: <br><br>

The cousin of the BR tag is <HR> or Horizontal Rule, which creates a segment break with a line.
<div style="width: 60%; margin: 0 auto;"> Divs can be given a specific width in pixels or an element-relative width in percentages. Here, within the style property, we set the width to 60% of the parent element. We also set the margin around the DIV to 0 on the top and bottom, and auto on the sides (which centers the DIV)

Easy CSS Code Reuse with Classes

CSS classes can be used to give Ps and DIVs and SPANs and lists consistent themes, and to save yourself re-typing the same properties set to the same values. These are often declared in STYLE sections. Inside this section CSS rules apply, like using : instead of = or ending each line with ;
These style sections can appear anywhere in the text, including at the bottom, and most browsers will still apply that style everywhere. To be safe, though, most often times STYLE sections are put in the HEAD section of the page, or at the top of the BODY section (but not inbetween). The STYLE section in the HTML source after this line of text is used to modify colors and borders in the elements that follow.
